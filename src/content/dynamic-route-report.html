Abstract

This project presents a dynamic route optimization system integrated with indoor mapping and Bluetooth Low Energy (BLE)-based user tracking for intelligent indoor navigation. Traditional navigation systems rely on static routes generated from initial user preferences, which fail to adapt to real-time user behavior. To address this limitation, the proposed system introduces an adaptive navigation framework that continuously updates routes based on user activity, location, and inferred interest.

The system combines multiple technologies, including indoor map development, spatial data processing, sensor-based activity recognition, and BLE-based positioning. The indoor map is initially designed using Adobe-based tools to create a visually accurate floor layout. This map is then processed using QGIS to generate structured spatial layers such as navigation paths, artifact locations, and BLE beacon positions. These layers are exported in GeoJSON format and integrated into a Flutter-based mobile application, where they are rendered using a layered visualization approach.

User location is estimated through BLE beacon signals by analyzing Received Signal Strength Indicator (RSSI) values. To improve accuracy, signal smoothing and weighted positioning techniques are applied. In addition, smartphone sensors such as accelerometers and gyroscopes are used to detect user activity, enabling the system to distinguish between movement and engagement behaviors.

The system employs a dynamic routing mechanism that updates navigation paths in real time by combining spatial distance with user preference data derived from observed behavior. By integrating indoor mapping, behavior analysis, and adaptive routing, the proposed solution enhances personalization, improves navigation efficiency, and provides a more interactive user experience in indoor environments such as museums and exhibitions.

Introduction

1.1 Background Literature

Indoor navigation has emerged as a critical research and application domain due to the increasing complexity of modern infrastructures such as museums, airports, shopping malls, hospitals, and large exhibition centers. Unlike outdoor environments where Global Positioning System (GPS) technology provides reliable and accurate positioning, indoor environments present significant challenges due to signal attenuation, multipath interference, and structural obstructions. As a result, alternative technologies and methods must be employed to support indoor positioning and navigation.

Over the past decade, various indoor navigation systems have been developed using technologies such as Wi-Fi fingerprinting, Radio Frequency Identification (RFID), Ultra-Wideband (UWB), computer vision, and Bluetooth Low Energy (BLE). Among these, BLE has gained widespread attention due to its low cost, energy efficiency, ease of deployment, and compatibility with smartphones. BLE beacon-based systems allow mobile devices to estimate proximity to specific locations by analyzing signal strength (RSSI), making them suitable for indoor navigation applications.

In parallel, the advancement of mobile computing and embedded sensors has enabled the integration of user behavior analysis into navigation systems. Smartphones are equipped with sensors such as accelerometers and gyroscopes, which can be used to detect user activities such as walking, standing, and sitting. These capabilities allow systems to go beyond simple location tracking and understand user interaction with the environment.

In museum environments, digital guides have evolved from static information systems to interactive applications capable of providing personalized recommendations and navigation assistance. Early systems relied heavily on predefined routes or manual user input, where visitors selected areas of interest before starting their visit. While these systems improved accessibility, they lacked adaptability and failed to respond to changes in user interest during the visit.

Recent research has highlighted the importance of context-aware and adaptive systems that can dynamically adjust recommendations based on real-time user behavior. For instance, dwell time near exhibits has been used as an indicator of user interest. However, dwell time alone can be misleading, as users may pause due to fatigue, crowding, or confusion rather than genuine interest. Therefore, combining multiple signals such as location, activity, and time is essential for accurate interest detection.

Furthermore, indoor mapping technologies have improved significantly with the use of Geographic Information Systems (GIS) tools such as QGIS. These tools enable the creation of structured spatial data, including routes, artifact locations, and beacon positions, which can be integrated into mobile applications. When combined with modern frameworks like Flutter, these systems can deliver highly interactive and visually rich user experiences.

Despite these advancements, most existing indoor navigation systems still rely on static route planning. They do not effectively incorporate real-time user behavior into route optimization. This limitation reduces the overall effectiveness of navigation systems, particularly in dynamic environments such as museums where user interests evolve continuously.

The present study addresses these limitations by proposing a system that integrates indoor mapping, BLE-based positioning, and sensor-based activity recognition to enable dynamic route optimization. By combining explicit user preferences with implicit behavioral data, the system aims to provide a more personalized and adaptive navigation experience.

1.2 Research Gap

Although significant progress has been made in indoor navigation and personalization technologies, several critical gaps remain in existing systems.

Firstly, most indoor navigation systems rely on static user profiles collected at the beginning of the visit. These profiles are used to generate predefined routes, assuming that user preferences remain constant. However, in real-world scenarios, user interests are dynamic and can change as they interact with different exhibits or environments.

Secondly, existing systems often treat location tracking and user behavior analysis as separate components. While BLE-based systems can estimate user position, they do not incorporate behavioral signals such as activity state or engagement level. Similarly, activity recognition systems do not utilize spatial context effectively. This lack of integration leads to incomplete understanding of user behavior.

Thirdly, many systems rely heavily on explicit user input, such as ratings or manual selections, to update preferences. This approach can be intrusive and may not accurately reflect actual user interest. Users are often unwilling to provide continuous feedback, making implicit sensing methods more desirable.

Another important gap lies in interest detection mechanisms. Many systems use simple metrics such as dwell time to infer interest. However, dwell time alone is insufficient, as it cannot distinguish between intentional engagement and incidental stopping. Without incorporating additional factors such as posture and proximity, these systems may produce inaccurate results.

Furthermore, indoor navigation systems frequently lack dynamic route adaptation capabilities. Even when user preferences are updated, the route is not recalculated in real time. This results in missed opportunities to guide users toward newly discovered areas of interest.

Finally, there is a lack of integrated frameworks that combine indoor mapping, BLE positioning, activity recognition, and route optimization into a single cohesive system. Most existing solutions address these components individually rather than as part of a unified architecture.

These gaps highlight the need for a system that can:

  • Continuously monitor user behavior
  • Accurately infer user interest using multiple signals
  • Dynamically update navigation routes
  • Provide seamless integration between mapping, sensing, and routing

1.3 Research Problem

The primary research problem addressed in this study is:

“How to design and implement a dynamic indoor navigation system that adapts routes in real time based on user behavior, location, and inferred interest?”

In traditional systems, navigation routes are generated based on initial user preferences and remain unchanged throughout the visit. This approach does not account for:

  • Changing user interests
  • Environmental factors (crowds, fatigue)
  • User interaction with exhibits

As a result, users may miss relevant content or follow inefficient routes, leading to a suboptimal experience.

Additionally, accurately detecting user interest in indoor environments is a complex challenge. Simple indicators such as location or dwell time are insufficient to capture true engagement. A more robust approach is required that combines multiple factors such as:

  • Activity state (walking, standing, sitting)
  • Proximity to artifacts
  • Duration of interaction

Another key challenge is achieving reliable indoor positioning. BLE-based systems provide approximate location estimates, but signal variability and environmental factors can affect accuracy. Therefore, efficient algorithms and filtering techniques are required to ensure stable positioning.

Moreover, dynamic route optimization must balance two competing objectives:

  1. Minimizing travel distance
  2. Maximizing user satisfaction based on preferences

Designing a system that effectively balances these objectives while maintaining a smooth and coherent navigation experience is a significant challenge.

1.4 Research Objectives

The main objective of this project is to develop a dynamic indoor navigation system that integrates indoor mapping, positioning technologies, user behavior analysis, and real-time route optimization. The system is designed to move beyond traditional static navigation approaches by incorporating adaptive mechanisms that respond to user interactions within the environment.

To achieve this goal, the study first focuses on developing a digital indoor map system using image-based coordinate models. This involves designing a visual indoor layout and preparing spatial data using tools such as QGIS. The spatial data is structured into multiple layers, including route paths, artifact locations, and BLE beacon positions, ensuring accurate representation and seamless integration with the mobile application.

Another key objective is the implementation of BLE-based indoor positioning. This includes deploying BLE beacons at strategic locations within the indoor environment and estimating the user’s position using Received Signal Strength Indicator (RSSI) values. To improve positioning accuracy, appropriate filtering and smoothing techniques are applied to reduce signal fluctuations and environmental noise.

The system also aims to detect user activity through smartphone sensors such as accelerometers and gyroscopes. By analyzing sensor data, the system classifies user behavior into activities such as walking, standing, and sitting. This activity information plays a crucial role in identifying meaningful user engagement with artifacts and supports the overall interest detection mechanism.

A central objective of the project is to enable dynamic route optimization. This is achieved by developing algorithms that continuously update navigation routes in real time based on user preferences and detected behavior. The routing mechanism combines distance-based calculations with preference-based weighting to ensure that navigation remains efficient while adapting to evolving user interests.

Finally, the system is designed to enhance the overall user experience by providing personalized navigation recommendations. By adapting routes based on real-time behavioral data, the system reduces the need for manual user input and offers a more intuitive and engaging interaction within indoor environments.

1.5 Scope of the Study

This study focuses on the design and implementation of a dynamic indoor navigation system specifically tailored for environments such as museums and exhibitions, where artifact-based navigation and user engagement are essential. The system is developed to operate within controlled indoor spaces, leveraging digital mapping and sensing technologies to deliver an adaptive navigation experience.

The scope of the study includes the creation of indoor maps using image-based coordinate systems, which allows accurate alignment between spatial data and visual representations. It also includes the implementation of BLE-based positioning to track user location within the environment. In addition, the system incorporates activity recognition using smartphone sensors to understand user behavior and integrates an interest detection mechanism that evaluates user engagement based on multiple conditions such as proximity, dwell time, and activity state.

Furthermore, the study covers the development of a dynamic route optimization framework using graph-based methods, enabling real-time updates to navigation paths. The implementation of a mobile application using Flutter is also within scope, ensuring that the system is accessible and functional on modern smartphones.

The system is designed primarily for single-user navigation scenarios and assumes the availability of BLE beacon infrastructure within the indoor environment. However, certain aspects are intentionally excluded from the scope of this study. These include outdoor navigation, high-precision localization requiring centimeter-level accuracy, multi-user interaction or crowd management features, and advanced artificial intelligence-based predictive modeling, which are considered potential areas for future research and development.

Methodology

The following methodology presents a revised and more detailed description of the system's operational logic. It extends the original research paper by adding explicit design rationale, decision criteria, and implementation guidance for each processing stage. The system is composed of seven tightly integrated components operating across two phases.

The overall system state at time t is formally defined as S_t = { P_t, A_t, L_t, I_t, R_t }, where P_t denotes the current user profile, A_t is the detected activity state, L_t is the estimated indoor location, I_t is the inferred interest state, and R_t is the active navigation route. Together, these five elements capture the full operational context of the system at any given moment during the visit.

Route adaptation is expressed as:

R_(t+1) = f( P_t, I_t, L_t )

where f(·) is the route update function that synthesizes the current profile, inferred interests, and visitor location to regenerate an optimized navigation path.

A. Survey-Based User Profiling

This component collects structured preference data from the visitor at the start of the museum session. It provides the cold-start input required to generate a meaningful initial navigation route before any behavioral data has been gathered.

The survey is administered through the mobile application immediately upon launch, with a target completion time of under 2 minutes. It collects two classes of data:

  • Visitor Type (u): Categorical – Student, Teacher, Professor, Civilian, Foreign Visitor.
  • Artifact Category Preferences (C₀): Multi-select from available museum categories (e.g., Coins, Clothing, Weapons, Statues, Sculptures, Ceramics, Manuscripts, Jewelry, Tools).

Formal definition of the initial profile:

P₀ = { u, C₀ }

The survey profile is explicitly treated as provisional. Visitors frequently have incomplete self-knowledge about their interests before encountering physical artifacts. The survey therefore provides a starting point expected to be refined and potentially overridden by observed behavior during the visit.

B. Activity Recognition for User Behavior Detection

The Activity Recognition Module (ARM) continuously classifies the visitor's physical activity into one of three states: walking, standing, or sitting. This classification is the primary behavioral signal used to distinguish transit between exhibits from engagement with an exhibit, and both from non-engagement rest states.

Sensor Input

The module reads six-dimensional inertial data from the smartphone's built-in sensors at each time step t:

x_t = [ a_x(t), a_y(t), a_z(t), g_x(t), g_y(t), g_z(t) ]

  • Accelerometer axes a_x, a_y, a_z: measured in m/s² – capture translational motion and gravitational component.
  • Gyroscope axes g_x, g_y, g_z: measured in rad/s – capture rotational motion and body orientation changes.

Window Segmentation & Feature Extraction

The continuous sensor stream is partitioned into fixed-length windows:

W_k = { x_t1, x_t2, …, x_tn }

For each window W_k the following features are extracted per axis, yielding an 18-dimensional feature vector: mean, variance, signal magnitude area (SMA), root mean square (RMS), and peak-to-peak amplitude.

Classification

The feature vector is passed to a trained supervised classifier that outputs:

A_hat_k ∈ { walking, standing, sitting }

Recommended classifiers: Random Forest (robust, fast inference), SVM with RBF kernel (strong generalization), or lightweight 1D-CNN (higher accuracy, more compute required).

The activity recognition module achieved an overall classification accuracy of approximately 93.4%, with the standing class reaching the highest reliability — a result that is particularly significant since interest detection depends critically on accurate identification of stationary behaviour. Walking and standing were clearly distinguishable due to their contrasting motion energy profiles, while sitting was identified by consistently low variance across both accelerometer and gyroscope channels. A recommended window size of 128 samples at 50 Hz yields a 2.56-second window with 50% overlap, providing a practical balance between temporal resolution and computational cost.

C. Artifact-Level Interest Inference

This component determines whether the visitor is genuinely interested in a specific artifact by fusing three independent signals: activity state, proximity to the artifact, and accumulated dwell time. This three-way gate is the core novelty of the system's interest detection approach.

Proximity Condition

An artifact is considered a candidate only when the estimated distance d_j(t) satisfies:

d_j(t) ≤ d_th (threshold range: 0.0 – 0.9 metres)

Dwell Time Condition

The visitor must remain near the artifact while standing for a minimum accumulated time:

T_j ≥ T_min (calibrated to 12 seconds)

Combined Interest Gate

I_j = 1 if A_hat_k = standing AND d_j(t) ≤ d_th AND T_j ≥ T_min

I_j = 0 otherwise

Sitting is explicitly excluded: museum visitors who sit are associated with rest or waiting behaviour. Including sitting would produce large numbers of false-positive detections near benches and rest areas.

D. Enhanced Interest Scoring

Rather than a binary decision, an enhanced interest score is computed for each artifact by combining multiple behavioral indicators. This reflects the fact that interest is often gradual rather than purely discrete.

Score_j = α·T_j + β·V_j + γ·Q_j

  • T_j – Valid dwell time (seconds): total accumulated standing time satisfying all gate conditions. Primary indicator of engagement depth.
  • V_j – Revisit count: number of distinct visits to artifact j. A returning visitor expresses stronger interest than a single-pass observer.
  • Q_j – Localization confidence: scalar [0, 1] derived from BLE signal quality. Down-weights scores recorded under poor positioning conditions.

The weighting coefficients must be normalized such that α + β + γ = 1.0. A recommended starting configuration assigns α = 0.50 to dwell time as the primary behavioral signal, β = 0.30 to revisit count as a reinforcement indicator, and γ = 0.20 to localization confidence as a quality gate. These values should be tuned empirically for each museum deployment, as the relative importance of each signal may vary with visitor demographics, exhibit layout, and BLE infrastructure quality.

This formulation is preferable to a simple standing-time count because a visitor may briefly stop near an object for reasons unrelated to interest, such as congestion or navigation hesitation. Incorporating revisit behavior and confidence reduces sensitivity to accidental pauses and noisy location estimates.

E. Category Preference Modeling

Each artifact in the museum is associated with a category such as clothes, coins, tools, jewelry, ceramics, sculptures, or manuscripts. Once artifact-level interest scores are computed, the system aggregates them at the category level to estimate the visitor's evolving preferences.

The cumulative category preference score is computed as:

Pref(C_m) = Σ Score_j for all a_j ∈ C_m

The resulting category scores are ranked in descending order, and the highest-ranked categories are treated as the visitor's current priorities. A re-routing event is triggered when:

  • The highest-ranked category changes identity (e.g., Weapons displaced by Coins), AND
  • The score differential exceeds a minimum threshold δ to prevent noise-induced route switching.

To illustrate this behavior, consider a visitor who initially selected Weapons as their preferred category during the pre-visit survey. During the actual visit, the system observed extended dwell times near Coins exhibits that were not part of the original selection. As the cumulative preference scores were updated, the Coins category rose above Weapons in the ranked list, followed by Clothing. This shift triggered a route adaptation in which the Coins section was inserted into the navigation path, ensuring the visitor was guided toward content that genuinely aligned with their emerging interests rather than their initial assumptions.

F. Dynamic Route Generation

The initial route is generated from the survey-based profile, but the navigation path is continuously updated as the visitor's observed preferences evolve. The museum is represented as a graph:

G = ( N, E )

  • N: Set of nodes representing rooms, corridors, or exhibit zones.
  • E: Set of walkable connections between nodes, weighted by traversal distance in metres.

A candidate route R = {n₁, n₂, …, nₖ} is evaluated by a cost function balancing path efficiency with preference alignment:

J(R) = Σᵢ d(nᵢ, nᵢ₊₁) + λ · Σᵢ φ(nᵢ)

  • d(nᵢ, nᵢ₊₁): Walking distance between consecutive nodes (metres).
  • φ(nᵢ): Penalty term – decreases when the node leads to a preferred category.
  • λ: Trade-off parameter balancing efficiency vs. personalization (recommended: 0.3–0.7).

The updated route is selected as:

R_(t+1) = argmin_{R ∈ R} J(R)

In practice this is solved with a modified Dijkstra or A* algorithm with preference-weighted edge costs. This formulation allows the system to introduce newly discovered interests into the route while maintaining a coherent path through the museum.

As a representative example, consider a visitor whose initial route covered Weapons followed by Clothing. During the visit, the system detected sustained engagement with Coins exhibits that fell outside this original path. Once the preference model ranked Coins as the top category, the route was dynamically updated to Weapons → Coins → Clothing, inserting the relevant section without disrupting the overall flow or significantly increasing walking distance.

G. BLE-Based Indoor Localization

Accurate identification of the nearest artifact requires a reliable estimate of the visitor's indoor location. The proposed system uses Bluetooth Low Energy beacons distributed throughout the museum. The mobile device scans nearby beacons and records their Received Signal Strength Indicator (RSSI) values in dBm.

RSSI Smoothing

Raw RSSI values are highly unstable due to obstacles, body blocking, and multipath effects. An Exponential Moving Average filter is applied:

r̃_t = η · r_t + (1 − η) · r̃_(t-1) (η recommended: 0.20 – 0.35)

RSSI-to-Distance Conversion

The smoothed RSSI is converted to an estimated distance using the log-distance path-loss model:

d_i = 10^( (r₀ − r̃_i) / (10·n) )

  • r₀: Calibrated RSSI at 1 metre (site-specific, typically −55 to −65 dBm).
  • n: Path-loss exponent (museum environments: n ≈ 2.0–3.5; requires site calibration).

Weighted Centroid Localization

Instead of trilateration, the system employs a weighted centroid approach that is computationally lighter and more stable under fluctuating RSSI conditions:

w_i = 1 / d_i

x̂ = Σ(w_i · x_i) / Σw_i ŷ = Σ(w_i · y_i) / Σw_i

Only the 3–4 strongest beacon signals are retained per estimation cycle to reduce the effect of distant and unstable measurements.

Position Smoothing & Outlier Rejection

p_t = ρ · p̂_t + (1 − ρ) · p_(t-1)

Position outliers (implied speed > 2 m/s) are rejected and the previous smoothed position is retained. The estimate is then map-matched to the nearest valid corridor segment to ensure displayed location remains consistent with navigable space.

H. Integrated System Workflow

The complete per-cycle data flow during the adaptive loop is as follows:

  1. Smartphone IMU samples x_t are collected at 50 Hz and buffered into windows.
  2. Activity Recognition classifies each window W_k → A_hat_k.
  3. BLE Localization produces smoothed position p_t and per-artifact distances d_j(t).
  4. Interest Inference evaluates the three-way gate for each candidate artifact j.
  5. Enhanced Scoring updates Score_j for all confirmed interest events.
  6. Preference Modeling aggregates scores to Pref(C_m) and checks the re-routing trigger.
  7. If triggered, Dynamic Route Generation computes the updated route R_(t+1).
  8. Updated route is pushed to the map UI for display to the visitor.

Results and Discussion

12.1 Overview of Evaluation

The proposed dynamic indoor navigation system was evaluated in a controlled indoor environment designed to simulate a museum setting. The evaluation aimed to analyze the performance of key system components, including activity recognition, BLE-based positioning, interest detection, and dynamic route optimization. The testing process focused on assessing both technical accuracy and system behavior under realistic user scenarios.

The system integrates multiple technologies, and therefore, its performance depends on the interaction between these components. For this reason, the evaluation was divided into four main aspects: activity recognition accuracy, indoor positioning reliability, effectiveness of interest detection, and the performance of dynamic route adaptation. These aspects collectively determine the overall usability and effectiveness of the system.

12.2 Activity Recognition Results

The activity recognition module plays a crucial role in identifying user behavior, which directly influences interest detection and route updates. The system utilizes smartphone sensors, specifically accelerometers and gyroscopes, to classify user activity into three primary categories: walking, standing, and sitting.

The results indicate that the activity recognition module achieved an overall accuracy of approximately 93%. Among the three activity states, standing was identified with the highest reliability. This is particularly significant because standing behavior is used as a primary indicator of user engagement with artifacts. Walking activity was also accurately detected due to its distinct motion patterns, characterized by higher variance in sensor readings. Sitting behavior, although less frequent, was successfully distinguished due to its low-motion characteristics.

The high accuracy of activity recognition ensures that the system can reliably differentiate between movement and engagement. This reduces the likelihood of false interest detection, where simple movement might otherwise be interpreted as user attention. The integration of accelerometer and gyroscope data proved effective in improving classification performance, particularly in distinguishing between stationary states such as standing and sitting.

However, it was observed that activity recognition performance may vary depending on how the user carries the device. For instance, holding the phone in hand, placing it in a pocket, or keeping it in a bag can affect sensor readings. Despite this variability, the system maintained acceptable performance levels for the intended application.

12.3 BLE-Based Indoor Positioning Results

Indoor positioning was evaluated using BLE beacons placed at strategic locations within the environment. The system estimates user location based on RSSI values obtained from nearby beacons. Due to the inherent instability of RSSI signals, smoothing techniques were applied to improve the consistency of position estimates.

The results show that the system achieved an average positioning accuracy in the range of approximately 0.6 to 1.2 meters. While this level of accuracy is lower than that of advanced positioning systems such as Ultra-Wideband (UWB), it is sufficient for identifying the user’s presence within specific zones or near particular artifacts. This level of precision is adequate for applications such as museum navigation, where zone-level accuracy is more important than exact coordinate positioning.

The use of a weighted centroid approach significantly improved positioning stability by combining signals from multiple beacons. Additionally, filtering techniques helped reduce sudden fluctuations in RSSI values caused by environmental factors such as obstacles, human movement, and signal interference.

Despite these improvements, some limitations were observed. BLE signals are highly sensitive to environmental conditions, and variations in signal strength can lead to temporary inaccuracies in position estimation. Factors such as crowd density, device orientation, and structural materials can influence signal propagation. Nevertheless, the system demonstrated sufficient robustness for practical indoor navigation applications.

12.4 Interest Detection Analysis

The interest detection mechanism is one of the most important components of the system, as it determines how user behavior influences route updates. The system uses a multi-condition approach, combining activity state, proximity to artifacts, and dwell time to infer user interest.

The results indicate that this multi-condition approach is significantly more reliable than using a single factor such as dwell time alone. For example, users who remained seated near an artifact for an extended period were not classified as interested, as the activity state indicated rest rather than engagement. Similarly, short pauses while walking were correctly ignored, preventing false positive detections.

In contrast, when users were detected standing near an artifact within a close proximity threshold and remained there for a sufficient duration, the system successfully identified this behavior as genuine interest. This demonstrates the effectiveness of combining spatial, temporal, and behavioral factors in interest detection.

The system also incorporates a scoring mechanism that accounts for dwell time, revisit frequency, and confidence levels. This allows the system to represent interest as a continuous measure rather than a binary decision. As a result, the system can rank artifacts and categories based on the strength of user engagement.

Overall, the interest detection mechanism proved to be robust and effective in distinguishing meaningful interactions from incidental behavior. This capability is essential for enabling accurate and meaningful route adaptation.

12.5 Dynamic Route Optimization Results

Dynamic route optimization is the core functionality of the system, and its performance was evaluated through scenario-based testing. The system continuously updates navigation routes based on user preferences and detected interests, ensuring that the route remains relevant throughout the visit.

The results demonstrate that the system successfully adapts routes in response to changes in user behavior. In one representative scenario, a user initially selected a specific category of artifacts, and the system generated a corresponding route. During the visit, however, the user showed repeated interest in a different category. The system detected this behavior and updated the route to include locations associated with the newly identified interest.

This ability to incorporate emerging interests into the navigation path is a significant improvement over traditional static routing systems. It ensures that users are guided toward relevant content even if it was not part of their initial preferences.

Another important observation is that the system maintains route coherence while adapting to changes. Instead of making abrupt or inefficient changes, the route optimization algorithm considers spatial constraints and minimizes unnecessary detours. This balance between adaptability and efficiency is achieved through the cost function, which combines distance and preference weighting.

The system also demonstrated stability in scenarios where no significant interest was detected. In such cases, the route remained unchanged, preventing unnecessary recalculations and ensuring consistent navigation behavior.

12.6 Discussion of System Performance

The overall performance of the system indicates that integrating multiple technologies—BLE positioning, activity recognition, and dynamic routing—can significantly enhance indoor navigation experiences. The combination of explicit user preferences and implicit behavioral data provides a more accurate and responsive understanding of user needs.

One of the key strengths of the system is its ability to operate in real time. By continuously monitoring user behavior and updating routes accordingly, the system creates a dynamic interaction between the user and the environment. This contrasts with traditional systems that rely on static data and limited user input.

Another important advantage is the use of lightweight technologies. The system does not require expensive infrastructure or complex hardware, making it suitable for practical deployment in real-world environments. The use of smartphones and BLE beacons ensures accessibility and cost-effectiveness.

However, several challenges remain. BLE signal variability continues to be a limitation, particularly in environments with high interference or complex layouts. Additionally, activity recognition accuracy may be affected by variations in device placement. These challenges highlight the need for further optimization and calibration.

12.7 Limitations

Although the system demonstrates promising results, certain limitations must be acknowledged. The accuracy of BLE-based positioning is influenced by environmental factors, and achieving consistent performance across different environments remains a challenge. Similarly, activity recognition depends on sensor data quality, which can vary based on user behavior and device usage patterns.

Another limitation is that the system primarily focuses on single-user scenarios. It does not currently account for group behavior, crowd dynamics, or multi-user interactions. Additionally, the system does not incorporate advanced predictive models that could anticipate user preferences before they are explicitly observed.

12.8 Summary

In summary, the results demonstrate that the proposed system effectively integrates indoor positioning, behavior analysis, and dynamic routing to enhance navigation experiences. The activity recognition module provides reliable input for interest detection, while BLE-based positioning offers sufficient accuracy for zone-level navigation. The multi-condition interest detection mechanism ensures robust identification of user engagement, and the dynamic routing algorithm successfully adapts navigation paths in real time.

The findings confirm that the proposed approach addresses key limitations of traditional indoor navigation systems and provides a foundation for further development and real-world deployment.

Conclusion

This study presented the design and implementation of a dynamic indoor navigation system that integrates indoor mapping, BLE-based positioning, activity recognition, and real-time route optimization. The primary objective of the research was to address the limitations of traditional static navigation systems by introducing an adaptive approach that continuously responds to user behavior and evolving interests.

The proposed system successfully combines multiple technologies into a unified framework. The indoor map, developed using image-based coordinates and spatial data prepared through QGIS, provides a structured and accurate representation of the environment. The use of GeoJSON for data storage and retrieval ensures efficient integration between the backend and the mobile application. The Flutter-based application effectively renders the map and associated layers, offering a user-friendly interface for navigation and interaction.

The implementation of BLE-based indoor positioning demonstrated that it is possible to achieve reliable zone-level location tracking using low-cost infrastructure. Although BLE signals are subject to environmental variability, the application of filtering and weighted estimation techniques helped improve positioning stability. The achieved accuracy was sufficient for practical navigation purposes within indoor environments such as museums.

A key contribution of this study is the integration of activity recognition using smartphone sensors. By classifying user behavior into walking, standing, and sitting, the system was able to distinguish between movement and meaningful engagement. This capability significantly enhanced the reliability of the interest detection mechanism. The use of a multi-condition approach, combining activity state, proximity, and dwell time, ensured that user interest was identified more accurately compared to traditional methods based solely on dwell time.

The dynamic route optimization component proved to be the most significant advancement of the system. By continuously updating routes based on real-time user data, the system was able to adapt navigation paths to reflect actual user preferences. This approach not only improved personalization but also ensured efficient navigation by balancing distance and interest-based factors. The ability to incorporate emerging interests during navigation represents a major improvement over conventional systems that rely on fixed routes.

The results of the study confirm that the integration of sensing technologies, spatial data, and adaptive algorithms can significantly enhance indoor navigation experiences. The system demonstrated strong performance in activity recognition, acceptable accuracy in indoor positioning, and effective real-time route adaptation. These outcomes indicate that the proposed approach is both practical and scalable for real-world applications.

Despite its strengths, the study also identified several limitations. BLE-based positioning is affected by signal instability and environmental conditions, which may impact accuracy in complex settings. Additionally, activity recognition performance can vary depending on device placement and user behavior. The system is currently designed for single-user scenarios and does not consider group dynamics or crowd interactions. Furthermore, the current implementation relies on rule-based interest detection, which may be further improved using advanced machine learning techniques.

Future work can focus on enhancing the robustness and capabilities of the system. This includes improving positioning accuracy through advanced sensor fusion techniques, such as combining BLE with pedestrian dead reckoning or machine learning-based correction models. The interest detection mechanism can be extended using probabilistic models or artificial intelligence to better capture user preferences. Additionally, incorporating contextual factors such as crowd density, time constraints, and environmental conditions can further improve route optimization.

Another potential direction is the expansion of the system to support multi-user environments, enabling collaborative navigation and crowd-aware recommendations. Integration with cloud-based analytics can also provide valuable insights into user behavior, supporting data-driven decision-making for organizations such as museums and exhibition centers.

In conclusion, this study demonstrates that dynamic indoor navigation systems based on real-time sensing and adaptive routing offer a promising solution for enhancing user experience in complex indoor environments. By moving beyond static navigation models and incorporating behavioral intelligence, the proposed system provides a more responsive, personalized, and efficient approach to indoor navigation. The findings of this research contribute to the growing field of smart environments and lay the foundation for future innovations in intelligent navigation systems.
